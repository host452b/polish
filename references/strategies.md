## A. 15 项纯文本策略（可直接写入单条提示词）

### 1. Prompt Repetition（提示重复）
- **模板**：将核心指令在提示中重复 2–3 次，或在不同位置重申关键约束。`请完成以下任务：{任务}` ＋ 末尾 `请再次确认并完成上述任务：{重复同一任务}`。
- **如何写进提示词**：在提示开头给出指令，结尾再用一句重申最关键的约束。
- **原理**：不依赖推理，重复输入同一指令可稳定提升输出对指令的遵循度，且基本不增加输出长度或延迟。
- **来源**：Google Research, 2024。
- **适合**：非推理类标准化任务、指令遵循、格式控制、文本分类、固定格式生成。

### 2. Option-First MCQ（倒装选项）
- **模板**：`以下哪一项正确？A. {A} B. {B} C. {C}　问题：{问题}`——选项前置。
- **如何写进提示词**：把候选项放在问题之前，让模型先锚定答案范围再补理由。
- **原理**：降低位置偏差，先确定答案空间再推理，提升选择题准确率。
- **来源**：Google, 2024。
- **适合**：选择题、知识测评、客观题判分、多选、分类。

### 3. Role-Rule Double Bind（角色+规则双保险）
- **模板**：`你是一位{领域}专家，严格遵守{规则}。若违反规则，立即回答"错误"并纠正自身。{任务指令}`
- **如何写进提示词**：一句设定专家角色（触发领域知识），一句设定可自检的硬规则（触发自我监督）。
- **原理**：角色触发知识检索，规则触发自我监督，双重降低错误率（宣称约 −18%）。
- **来源**：Microsoft, 2025。
- **适合**：专业问答、合规内容、标准化报告、高精度输出。

### 4. Self-Contrast（反向验证）
- **模板**：`请回答：{问题}。要求：1) 先给明确答案；2) 列举 3 个可能推翻该答案的证据并各自评可信度(1–5)；3) 若证据总分 > 0，则据此修正答案。`
- **如何写进提示词**：在"给答案"之后强制追加"自我反驳 + 据证据修正"的步骤。
- **原理**：强制自我反驳、主动排查错误，显著降低幻觉（宣称约 −24%）。是单提示场景下 `Self-Consistency（自洽性检查）` 的替代。
- **来源**：Meta, 2025。
- **适合**：事实问答、学术内容、谣言甄别、高风险决策、减少幻觉。

### 5. Symbolic Placeholder（符号占位符）
- **模板**：`{任务指令}。要求：必须将最终答案放在 ▶︎▶︎answer▶︎▶︎ 之间，否则判为无效输出。`
- **如何写进提示词**：规定一个唯一锚点标记包裹最终答案。
- **原理**：给出明确答案锚点，提升答案提取精度（宣称 92%→98%）。
- **来源**：Anthropic, 2025。
- **适合**：信息抽取、精准提答、批量结果汇总、结构化输出、自动化解析、API 集成。

### 6. Knowledge Time-Stamp（知识时间戳）
- **模板**：`截至 2025-01，{背景知识}。若涉及该时间点之后的新信息，请标注"可能已过期"。{任务指令}`
- **如何写进提示词**：显式写出知识截止时间，并要求对其后信息加不确定标注。
- **原理**：显式校准知识截止时间，避免使用过期知识（宣称降幻觉约 −15%）。
- **来源**：Stanford, 2025。
- **适合**：时效性问答、行业动态分析、新闻生成、动态知识查询。

### 7. Layered Prompt（分层提示）
- **模板**：① 目标（1 句明确任务目标）② 规则（1 行核心执行规则）③ 示例（数行参考示例）④ 格式（1 句明确输出格式）。
- **如何写进提示词**：按 目标→规则→示例→格式 四层分块组织全部指令。
- **原理**：匹配人类最低认知负荷的信息结构，提升对复杂指令的解析一致性。
- **来源**：UIUC, 2024。
- **适合**：多要求复杂指令、跨维度生成、定制方案、长文本、多约束任务。

### 8. Mirror Test（镜子测试）
- **模板**：`{任务指令}。现在请你再读一遍刚才的回答，你会修改哪一句？请只输出修改后的整句。`
- **如何写进提示词**：在任务末尾追加一次"自我复读 + 改写最弱一句"的自检。
- **原理**：低成本触发二次自检与反思（宣称 F1 约 +2.3）。
- **来源**：Tsinghua, 2025。
- **适合**：文本校对、回答优化、细节打磨、最终质检、迭代改进。

### 9. Flipped Interaction（翻转交互）
- **模板**：`执行任务前，请先澄清以下 3 个问题：1){任务边界}；2){输出格式}；3){核心需求}。澄清完成后，再据我的回复执行 {任务指令}。`
- **如何写进提示词**：让目标模型在动手前先向用户提出关键澄清问题。
- **原理**：前置需求澄清，减少因指令模糊/歧义导致的偏差。
- **来源**：OpenAI / 社区实践, 2023。
- **适合**：需求模糊的定制任务、个性化生成、复杂项目、意图不明场景。
- **注意**：本 skill 的输出契约仍是"只给一个代码块，内含 English Prompt 和中文提示词"——澄清环节写进**双语重构提示词内部**，由目标模型对终端用户发起，而不是你回头问下达任务的人。

### 10. Chain-of-Thought (CoT)（链式思考）
- **模板**：`请解决：{问题}。要求：think step by step，逐步推导并给出每一步的理由。`
- **如何写进提示词**：显式要求分步推理并展示中间过程。
- **原理**：激活中间推理能力，显著提升复杂任务准确性；提示工程最有影响力的技术之一。
- **来源**：Google, 2022 (Wei et al.)。
- **适合**：数学计算、逻辑分析、多步骤问题求解。

### 11. Prompt Chaining（提示链）
- **模板**：`【步骤1】完成子任务1 → 输出 A；【步骤2】基于 A 完成子任务2 → 输出 B；【步骤3】基于 B 完成最终任务。`
- **如何写进提示词**：把复杂任务拆成有序子步骤，并明确"前一步输出作为后一步输入"——单条提示内顺序执行即可，不必真的多次调用。
- **原理**：拆解 + 上下文累积提升整体准确性。
- **来源**：LangChain / 社区实践, 2023。
- **适合**：长文摘要、多步数据分析、复杂报告、多阶段工作流、文档流水线。

### 12. Meta-Prompting（元提示词）
- **模板**：`请针对以下会反复执行的任务：{任务描述}，先生成一个优化的提示词（要求能提升执行效率与准确性），然后用它执行该任务。`
- **如何写进提示词**：让模型先自产一个更优提示词，再据其执行。
- **原理**：模型自我分析、自我优化提示，降低人工调优成本。
- **来源**：Stanford / 社区研究, 2024。
- **适合**：重复执行的标准化任务、批量文本处理、提示词自动调优。

### 13. Tree of Thoughts (ToT)（思维树）
- **模板**：`请解决：{问题}。要求：1) 列出 3 种推理路径；2) 评估每条可行性(1–10)；3) 选最优路径分步推导；4) 若推导失败，切换次优路径重试。`
- **如何写进提示词**：要求显式生成多条思路分支、评估、择优、可回溯。
- **原理**：突破线性思维，多路径探索与回溯。
- **来源**：Princeton / DeepMind, 2023 (Yao et al.)。
- **适合**：高难推理、决策分析、复杂规划、需要探索多种可能的任务。

### 14. ReAct (Reasoning + Acting)
- **模板**：`请按「思考-行动-观察」循环完成任务。思考：当前要做什么？行动：执行具体操作（检索/调用工具）。观察：结果是否符合预期？是否需要调整下一步？`
- **如何写进提示词**：规定 思考→行动→观察 的循环结构。
- **原理**：推理与行动交替，依观察动态调整策略；构建 Agent 的核心框架。
- **来源**：Google / Princeton, 2023 (Yao et al.)。
- **适合**：信息检索辅助问答、数据查询、工具调用类任务、Agent。
- **注意**：纯文本可写出该结构，但"行动"在**有工具/检索可用**时才真正发挥作用；纯对话场景下它退化为结构化推理。

### 15. Step-Back Prompting（退一步提示）
- **模板**：`在解答具体问题前，请先退一步：① 这道题属于哪一类问题、背后的通用原理 / 定律 / 公式是什么？先把它写出来；② 再用该原理一步步代入、解出具体问题。问题：{具体问题}`
- **如何写进提示词**：强制"先抽象（让模型自己说出通用原理 / 类别）→ 再具体（代入求解）"两段式；关键是让**目标模型自行定位框架**，而不是把原理硬塞给它。
- **原理**：先把注意力锚定到正确的高层概念 / 框架上，再做具体推理，可减少被题面细节带偏、用错方法。与 CoT 互补——CoT 管"逐步推"，Step-Back 管"先选对框架"；与思维树不同——ToT 横向铺多条路径，Step-Back 纵向先升一层抽象。
- **来源**：Google DeepMind, 2023（Zheng et al., "Take a Step Back: Evoking Reasoning via Abstraction"）。
- **适合**：物理 / 化学 / 数学定理应用题、需先定位正确原理或类别的知识推理、容易被表面细节误导的题。

## B. 任务类型 → 策略组合（纯文本版 · 含改动说明）

| 任务类型 | 推荐组合 | 核心优势 | 相对原始资料的改动 |
|---|---|---|---|
| 选择题 / 客观判定 | Option-First MCQ（倒装选项） + Symbolic Placeholder（符号占位符） + Self-Contrast（反向验证） | 降低位置偏差 + 精准提取 + 自我排错 | Self-Consistency → Self-Contrast |
| 专业领域 / 合规生成 | Role-Rule Double Bind（角色+规则双保险） + Knowledge Time-Stamp（知识时间戳） + Mirror Test（镜子测试） | 专业检索 + 时效校准 + 二次优化 | 不变 |
| 复杂推理求解 | Chain-of-Thought (CoT)（链式思考） + Tree of Thoughts (ToT)（思维树） + Self-Contrast（反向验证） | 分步 + 多路径 + 自我反驳 | 不变 |
| 原理 / 框架型推理（物理化学、数学定理） | Step-Back Prompting（退一步提示） + Chain-of-Thought (CoT)（链式思考） + Self-Contrast（反向验证） | 先锁定正确原理 + 分步 + 自我排错 | 新增（Step-Back Prompting，社区常用，原始资料未含） |
| 批量标准化任务 | Meta-Prompting（元提示词） + Layered Prompt（分层提示） + Prompt Repetition（提示重复） | 自优化 + 清晰解析 + 稳定遵循 | Dynamic Few-Shot → Prompt Repetition |
| 创意与事实融合 | Prompt Chaining（提示链） + Self-Contrast（反向验证） + Mirror Test（镜子测试） | 分阶段发散收敛 + 事实自查 + 润色 | Temp-Decay Sampling/ReAct → Prompt Chaining+Self-Contrast+Mirror Test（有检索工具时 Self-Contrast 可换 ReAct） |
| 高精度问答 | Role-Rule Double Bind（角色+规则双保险） + Self-Contrast（反向验证） + Mirror Test（镜子测试） | 三重保险降错误率 | Self-Consistency → Self-Contrast |
| 结构化输出 | Layered Prompt（分层提示） + Symbolic Placeholder（符号占位符） + Prompt Repetition（提示重复） | 确保格式一致性 | 不变 |
| 减少幻觉 | Knowledge Time-Stamp（知识时间戳） + Self-Contrast（反向验证） + Mirror Test（镜子测试） | 多层事实核查 | 不变 |
| Agent / 工具调用 | ReAct + Prompt Chaining（提示链） | 系统化处理多步骤 | 不变（ReAct 需工具） |
| 需求模糊 | Flipped Interaction（翻转交互） → Layered Prompt（分层提示） →（提示词内）执行 | 先澄清后执行 | 不变 |

## C. 被排除的 3 项策略（需提示词之外的能力）

这些不能塞进单次模型调用的双语提示词，仅当用户掌握 **API/解码控制或多轮编排**时，可作为"设置与流程建议"提供（需先与用户确认，超出默认输出契约）。

### Dynamic Few-Shot（动态少样本，CMU, 2024）
- **原始模板**：先用 0-shot 执行任务，提取第一轮输出中的正确部分作为示例 1–3，再基于这些动态示例重新执行。
- **为何排除**：需要"先跑一轮、抽取自身输出、再跑第二轮"，本质多轮。
- **适合**：零样本效果差的垂直领域分类、专业术语标注、冷启动。

### Self-Consistency（自洽性检查，Google / UW, 2023, Wang et al.）
- **原始模板**：对同一任务生成多个独立答案，选共识度最高者为最终输出。
- **为何排除**：经典做法需多次独立采样后投票。单提示场景请改用 `Self-Contrast（反向验证）` 获得近似收益。
- **适合**：模糊需求生成、多解问题、数学推理、逻辑判断、开放式问答。

### Temp-Decay Sampling（温度衰减解码，Google DeepMind, 2024）
- **原始模板**：前 50% token 用较高温度（如 0.7）保创意，后 50% 自动降到较低温度（如 0.3）保事实。
- **为何排除**：逐 token 调温属解码参数，模型无法在自身提示里自行改采样温度。
- **适合**：创意文案、商业报告、需要平衡创新与准确的任务。
- **纯文本近似**：用「提示链」分阶段（先发散后收敛）+「反向验证」收敛事实，可在不碰解码参数的前提下逼近其意图。
